# Documentation standards mapping

This repository documentation is written to align with GitHub Docs best practices:

- Align content to user needs
- Structure content for readability
- Write for readability (plain language, active voice, concise phrasing)
- Format for scannability

Reference:
- https://docs.github.com/en/contributing/writing-for-github-docs/best-practices-for-github-docs

## How this repo applies those standards

- Each primary doc starts with purpose/context before details.
- Setup and operations docs are task-oriented and ordered by user workflow.
- Long sections use short paragraphs, lists, and descriptive headings.
- Security-sensitive guidance emphasizes safe defaults and explicit opt-in behavior.
- Template language avoids organization-specific assumptions.

## Ongoing documentation checklist

When editing docs, verify:

1. Audience and purpose are clear in the first section.
2. Steps are ordered by user workflow.
3. Headings are specific and descriptive.
4. Sentences are concise and use active voice.
5. Important variables/files are easy to locate.
6. Security implications are called out clearly.
7. Related docs are linked when readers need follow-up context.
